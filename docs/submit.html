Submit (Demo)

GitHub Pages is static, so this demo does not accept file uploads. For a fastest MVP, we recommend a GitHub-based workflow: participants submit via Pull Request (PR) or GitHub Issue, and maintainers run scoring and update the leaderboard JSON.

Option A — Submit via Pull Request (recommended for demo)

1) Fork this repository

2) Add your files under submissions/<team>/<YYYY-MM-DD>/

3) Add/update docs/data/leaderboard.json only if you are an admin (maintainers will do this after scoring)

4) Open a PR

Maintainers run your code in a sandbox and post results, then merge a leaderboard update.

Option B — Submit via GitHub Issue (simplest)

Create a GitHub Issue titled [Submission] Model Name and attach or link:

• model script (Python/R)

• dependencies file (optional)

• methodology PDF (optional)

• weight file / forecast file (if required by track)

This is not secure for restricted data; it’s only for a pilot / demo.

Full platform (later)

To match a production CTF system, you’ll typically add:

• authenticated upload portal (S3 / object storage)

• job queue (Redis/Celery) + sandbox runner (Docker/Apptainer)

• scoring service writing metrics to Postgres

• an admin panel for review/audit
