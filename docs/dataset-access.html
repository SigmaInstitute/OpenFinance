Dataset Access (Demo)

In many finance benchmarks, raw vendor data cannot be redistributed. A common solution is a WRDS-gated distribution of standardized derived tables + a “model-to-data” evaluation server that keeps the holdout test set private.

Recommended approach

WRDS-gated tables Provide scripts and a data dictionary for participants to extract standardized tables from WRDS (or a partner’s licensed data platform).

Private test set Keep the holdout test set on the evaluation server. Participants submit code/models; the platform executes them next to the data.

Versioning Publish dataset version numbers (e.g., v0.1, v0.2) and pin train/validation splits.

Example: table schema

features: (date, asset_id, x1, x2, ..., xP) returns: (date, asset_id, ret) liquidity: (date, asset_id, adv, spread_proxy, amihud, ...)

This demo does not ship data. It only demonstrates the website shell and leaderboard rendering.
