Rules

OpenFinance follows a Common Task Framework (CTF) philosophy: strict temporal integrity, standardized inputs/outputs, and reproducible submissions.

Rule 1 — Temporal integrity

At time t, your model may only use information available at or before t. No look-ahead, no revisions, no peeking at future characteristics or returns.

Rule 2 — Algorithmic feature selection

Any feature engineering or feature selection must be programmatic and determined only from training data (or past data within a rolling window). Manual cherry-picking based on knowledge of future performance is not allowed.

Rule 3 — Data sources

Only the benchmark dataset is permitted. External data sources (macro indicators, alternative data, web-scraped signals, proprietary datasets) are prohibited unless the track explicitly allows them.

Rule 4 — Reproducibility

Submissions must be fully reproducible:

• Python: provide requirements.txt or pyproject.toml with pinned versions

• R: provide renv.lock

In the full platform, code runs in a network-disabled sandbox (e.g., Docker/Apptainer) with time/memory limits.

Rule 5 — Implementability (Track A)

Implementable Asset Pricing is evaluated net of transaction costs under a standardized cost model and liquidity constraints. Participants may not override the platform’s cost accounting.
